About maintaining this Contributing Guide itself.

What to add?

For this contrib guide to stay useful it needs to be reasonably complete, easy to navigate and concise. So we don't need to mention obvious things.

Try to include examples, an explaination of why, and screenshots if it helps.

If there is an open question the team must decide on, consider adding a todo:

TODO: decide upon the color of our uniforms

Getting started locally

git clone ssh://git@github.com:DatapuntAmsterdam/contrib-guide.git
cd contrib-guide
pip install virtualenv
virtualenv .
./bin/pip install -r requirements.txt
make docserver

This starts an HTTP-server on port 8000. All changes you make in the docs directory are visible there immediately.

To make a change

  1. Create a branch
  2. Edit the docs locally
  3. Commit and push the branch
  4. Create a pull request
  5. After review, merge the branch into master
