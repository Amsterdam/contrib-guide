Introduction

This guide covers everything you need to succesfully start contributing to Datapunt. It formalizes coding styles, procedures, and all other things we agree (to disagree) upon.

Referring to this guide

Please refer to this document whenever applicable, eg. while performing code reviews and acceptance tests. If there are differing opinions about which direction to take or style to use, you can refer to a section in this document. For example, a comment might look like:

There is a missing docstring. We agreed that all new classes will contain at least a one liner. See section docstrings in the style guide.

The canonical URL of this documentation is https://datapuntamsterdam.github.io/contrib-guide/

Contributing to this guide

What to add?

For this contrib guide to stay useful it needs to be reasonably complete, easy to navigate and concise. So we don't need to mention obvious things.

Try to include examples, an explaination of why, and screenshots if it helps.

If there is an open question the team must decide on, consider adding a todo:

TODO: decide upon the color of our uniforms

Getting started locally

git clone ssh://git@github.com:DatapuntAmsterdam/contrib-guide.git
cd contrib-guide
pip install virtualenv
virtualenv .
./bin/pip install -r requirements.txt
make docserver

This starts an HTTP-server on port 8000. All changes you make in the docs directory are visible there immediately.

To make a change

  1. Create a branch
  2. Edit the docs locally
  3. Commit and push the branch
  4. Create a pull request
  5. After review, merge the branch into master
